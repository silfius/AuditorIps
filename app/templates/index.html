    Configuración del host — Routing e iptables

    Para que el host pueda escanear la red secundaria por la interfaz correcta (p.ej. wlp2s0 hacia 192.168.18.0/24), el sistema operativo necesita dos cosas: una ruta y una regla de firewall. Ejecuta los siguientes comandos como root en el host (fuera del container Docker):

    1. Ruta de red (temporal — hasta reinicio) 
    # Sustituye los valores por los de tu entorno:
IFACE=wlp2s0            # interfaz que conecta a la red secundaria
NET=192.168.18.0/24     # CIDR de la red secundaria
GW=192.168.18.1         # puerta de enlace en esa red (normalmente el router ISP)

ip route add $NET dev $IFACE src $(ip -4 addr show $IFACE | grep -oP '(?<=inet )\S+(?=/)') 2>/dev/null || \
ip route replace $NET dev $IFACE
    2. Permitir tráfico nmap/ICMP saliente (iptables) 
    # Permitir ICMP (ping sweep) y TCP SYN desde el container hacia la red secundaria
IFACE=wlp2s0
NET=192.168.18.0/24

iptables -A OUTPUT -o $IFACE -d $NET -p icmp -j ACCEPT
iptables -A OUTPUT -o $IFACE -d $NET -p tcp --syn -j ACCEPT
iptables -A INPUT  -i $IFACE -s $NET -p icmp -j ACCEPT
iptables -A INPUT  -i $IFACE -s $NET -p tcp  -j ACCEPT

# Verificar que las reglas están activas:
iptables -L OUTPUT -n -v | grep $NET
    3. Persistencia tras reinicio (opcional) 
    # Ubuntu/Debian — guardar reglas iptables actuales:
apt install -y iptables-persistent
netfilter-persistent save

# Ruta persistente — añadir a /etc/network/interfaces o crear archivo en:
# /etc/NetworkManager/dispatcher.d/   (si usas NetworkManager / Ubuntu Desktop)
# /etc/systemd/network/               (si usas systemd-networkd)

# Ejemplo ruta persistente con NetworkManager (crear fichero):
cat > /etc/NetworkManager/dispatcher.d/99-secondary-route <<'EOF'
#!/bin/bash
[ "$1" = "wlp2s0" ] && [ "$2" = "up" ] && \
  ip route add 192.168.18.0/24 dev wlp2s0 2>/dev/null
EOF
chmod +x /etc/NetworkManager/dispatcher.d/99-secondary-route
    4. Verificar conectividad desde el container 
    # Desde el host (el container usa network_mode: host):
ping -c2 -I wlp2s0 192.168.18.1   # debe responder el router ISP

# Nmap de prueba con la interfaz:
nmap -sn -e wlp2s0 192.168.18.0/24

# Si nmap no encuentra nada, verificar que la ruta existe:
ip route get 192.168.18.1
    Nota Docker: El container ya usa network_mode: host y cap_add: [NET_RAW, NET_ADMIN], por lo que nmap puede acceder directamente a cualquier interfaz del host. No se requieren cambios en docker-compose.yml.
